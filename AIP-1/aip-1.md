## Summary
<!--"If you can't explain it simply, you don't understand it well enough." Provide a simplified and layman-accessible explanation of the AIP.-->

An AIP (Agora Improving Proposal) is a design document to describe a new feature, a new project or a new initiative, etc. The AIP should provide a concise specification and a rationale of the design. The AIP author is responsible for building consensus and implementing together with the community.

## Abstract
<!--A short (~200 word) description of the technical issue being addressed.-->

AIPs Guidelines is a mini playbook for the contributors to convert ideas into design documents for new features, new projects, new initiatives and more for the Agora community. We encourage the contributors and collaborators to submit AIPs with a templated format, follow up the ongoing procedures in time, and implement the approved AIP on time in a planned way.

## Specification
<!--The technical specification should describe the syntax and semantics of any new feature. The specification should be detailed enough to allow competing, interoperable implementations for any of the current Agora platforms.-->

We'd specify the AIP Guidelines step by step:

1. Read the [README](https://github.com/agorahub/AIPs/blob/master/README.md) and the [AIP guidelines](https://github.com/agorahub/AIPs/blob/master/AIP-1/aip-1.md) herein as a playbook.

2. Study the previous [AIPs](https://github.com/agorahub/AIPs/blob/master) to generate ideas and check availability.

3. Draft your AIP using the [AIP template](https://github.com/agorahub/AIPs/blob/master/aip-template.md).

4. Fork and Pull Request the [AIPs repository](https://github.com/agorahub/AIPs).

5. Commit and Comment to your Pull Request for Review and further proceeding.

6. Collaborate with the Agora community for implementation and project management.
